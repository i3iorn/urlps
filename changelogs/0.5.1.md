# urlps 0.5.1

Release date: 2026-06-15

## Highlights

- Added a focused patch release that improves reliability through expanded edge-case and failure-path testing.
- Increased coverage across parser, security, validation, URL object behavior, and package facade helpers.
- Strengthened confidence in secure defaults and defensive error handling without changing public APIs.

## Added

- New coverage-focused test suites:
  - `tests/test_coverage_gaps.py`
  - `tests/test_coverage_gaps2.py`
- Additional tests for parser edge cases, including:
  - IDNA encoding failures in host parsing
  - query parsing error paths
  - scheme/port default and invalid-port flows
- Additional tests for security checks, including:
  - DNS rebinding error and rate-limit paths
  - parser confusion patterns
  - canonicalization and IPv6 edge handling
  - phishing/punycode heuristic branches
- Additional tests for validation and URL helpers, including:
  - host and IPv4 validation edge cases
  - cache introspection/clearing paths
  - URL copy/override guards and string rendering behavior

## Changed

- Package version bumped to `0.5.1`.
- Internal quality gates now cover more exception-handling and fallback branches.
- Documentation changelog URL should point to `changelogs/0.5.1.md` for this release.

## Security Notes

- No new security features were introduced in this patch release.
- Existing security controls received broader regression coverage for malformed input and network-failure scenarios.
- This release reduces risk of silent behavior drift in security-sensitive code paths.

## Migration Notes

- No migration required.
- No breaking API changes expected from `0.5.0` to `0.5.1`.
