# urlps 0.5.0

Release date: 2026-05-22

## Highlights

- Introduced policy-based security controls via `SecurityPolicy` presets (`strict`, `balanced`, `internal`)
- Added structured security findings and validation interfaces for better testability and diagnostics
- Added structured audit event callbacks with optional correlation IDs and safer URL redaction
- Added typed error codes to improve downstream error handling and observability
- Added secure builder flows (`build_secure`, `Builder.compose_secure`) and stricter mutation-time validation

## Added

- `urlps.security_policy.SecurityPolicy`
- `urlps.security_policy.resolve_security_policy`
- `parse_url(..., policy=..., correlation_id=...)`
- `parse_url_unsafe(..., policy=..., correlation_id=...)`
- `build_secure(...)`
- `Builder.compose_secure(...)`
- `URL.validate(...)`
- `URL.redacted()`
- `URL.security_findings`
- Structured audit APIs:
  - `set_audit_event_callback(...)`
  - `get_audit_event_callback()`
- Typed error code enum:
  - `urlps.exceptions.ErrorCode`

## Changed

- Package version bumped to `0.5.0`
- `parse_url()` now uses policy-based security evaluation (default policy: `balanced`)
- Security checks are consistently re-applied when mutating URL objects through `copy()` and `with_*()` helpers
- DNS rebinding checks now include deterministic status reporting internally and retry/backoff controls
- Audit callback invocation can redact sensitive URL data (userinfo and sensitive query parameters)
- Project metadata changelog URL now points to `changelogs/0.5.0.md`

## Security Notes

- Added configurable controls for:
  - query injection detection
  - dangerous port blocking
  - credential rejection in URL userinfo
  - canonical URL enforcement
- Improved DNS check behavior with clearer failure classification and safer defaults

## Migration Notes

Most existing code should continue to work with default `balanced` policy.

If you need maximum hardening, explicitly use:

```python
from urlps import parse_url

url = parse_url("https://example.com", policy="strict")
```

If you need trusted/internal behavior:

```python
from urlps import parse_url_unsafe

url = parse_url_unsafe("http://localhost:3000", policy="internal")
```
