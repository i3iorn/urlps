# urlps 0.6.0

Release date: 2026-05-25

## Highlights

- Standardized the public import path on `urlps` across packaging, docs, tests, and performance scripts.
- Strengthened DNS rebinding policy behavior with explicit, configurable connect error handling.
- Updated `parse_url_unsafe()` to honor an explicit `policy` object exactly.

## Added

- Policy field `dns_fail_open_on_connect_error` for DNS connection-check behavior control.
- DNS API arguments in `dns_guard` for explicit fail-open/fail-closed connection handling.
- Import-path regression coverage (`tests/facade/test_import_path.py`).
- Search hygiene support via a new `.rgignore` file to reduce noise from generated/IDE artifacts.

## Changed

- Package import path in code examples and tests now uses `urlps` instead of `src.urlps`.
- `SecurityPolicy.strict()` defaults to DNS connect **fail-closed** (`dns_fail_open_on_connect_error=False`).
- `SecurityPolicy.balanced()` defaults to DNS connect **fail-open** (`dns_fail_open_on_connect_error=True`).
- `parse_url_unsafe(policy=...)` now preserves all values from the provided policy.
- Package version bumped to `0.6.0`.

## Security Notes

- DNS verification behavior is now explicitly policy-controlled rather than implicit.
- Strict mode now favors security on DNS connect verification failures by default.
- Balanced/internal modes retain availability-friendly defaults unless overridden.

## Migration Notes

- Replace imports like `from src.urlps import ...` with `from urlps import ...`.
- If your deployment expects fail-open DNS connect behavior in strict mode, set:
  - `SecurityPolicy.strict(dns_fail_open_on_connect_error=True)`
- If you relied on `parse_url_unsafe(..., policy=...)` implicitly disabling phishing checks, set the policy explicitly to disable phishing.
